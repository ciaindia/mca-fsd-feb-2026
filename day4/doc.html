Docker Made Simple

A visual guide to containerizing Node.js applications

📦 Core Docker Concepts

The Dockerfile

Think of it as a recipe. It contains the instructions to build your Docker Image.

The Image

The blueprint. A snapshot of your application and its entire environment (OS, libraries, code).

The Container

The living application. A running instance of an image. You can run many containers from one image.

🌪️ The Docker Workflow

Write Code → Build Image → Created Blueprint → Start Container

🚀 Sample Node.js App

Our app consists of two main files: the logic and the Docker recipe.

1. index.js (The App)

const express = require('express');
const app = express();
const port = 3456;

app.get('/', (req, res) => {
  res.send('
Hello from Docker!
');
});

app.listen(port, '0.0.0.0', () => {
  console.log(`App running on port ${port}`);
});

2. Dockerfile (The Recipe)

FROM node:18-alpine
WORKDIR /usr/src/app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3456
CMD [ "node", "index.js" ]

🛠️ Docker Cheat Sheet

  • Build the Image
    docker build -t my-app .

    Creates an image named 'my-app' using the current directory.

  • Run the Container
    docker run -p 3000:3456 my-app

    Starts a container. Maps host port 3000 to container port 3456.

  • List Running Containers
    docker ps
  • Stop & Remove
    docker stop [NAME]
    docker rm [NAME]
