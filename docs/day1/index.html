üî§ JavaScript Basics

Variables and Constants

// Variables can be reassigned let x = 10; console.log(`X = ${x}`); // Constants cannot be reassigned const PI = 3.14; console.log(`PI = ${PI}`);

Key Differences:

Feature let const var (old)

Reassignable ‚úÖ Yes ‚ùå No ‚úÖ Yes

üí° Best Practice: Always use const by default. Only use let when you know the value will change. Avoid var in modern JavaScript.

‚ö° Functions in JavaScript

Three Ways to Define Functions

1. Function Declaration

function getSqr(x) { console.log(x * x); } getSqr(5); // Output: 25

2. Function Expression

const getSqr = function (x) { console.log(x * x); } getSqr(6); // Output: 36

3. Arrow Function (ES6+) ‚≠ê

const getSqr = (x) => { console.log(x * x); } getSqr(7); // Output: 49
Arrow Functions Benefits:
  • Concise syntax
  • Implicit return for single expressions
  • Lexical this binding
  • Modern and widely used in React

Higher-Order Functions & Callbacks

graph TD A[Higher-Order Function] -->|Receives| B[Function as Parameter] A -->|Executes| C[Callback Function] C -->|Performs| D[Specific Operation] style A fill:#667eea,stroke:#333,stroke-width:2px,color:#fff style B fill:#764ba2,stroke:#333,stroke-width:2px,color:#fff style C fill:#f093fb,stroke:#333,stroke-width:2px style D fill:#4facfe,stroke:#333,stroke-width:2px
const add = (x, y) => { console.log(x + y); } const sub = (x, y) => { console.log(x - y); } // Higher-order function that accepts a callback const calc = (a, b, op) => { op(a, b) // Execute the callback function } calc(5, 6, add); // Output: 11 calc(5, 6, sub); // Output: -1
Callback Functions:
  • Functions passed as arguments to other functions
  • Enable code reusability and abstraction
  • Foundation for asynchronous programming
  • Commonly used in array methods, event handlers, and timers

üìä Array Methods

Array Manipulation

Basic Array Iteration

let nums = [1, 2, 3, 4, 5]; let sum = 0; for(let i = 0; i < nums.length; i++) { sum = sum + nums[i]; } console.log(sum); // Output: 15

map() - Transform Array Elements

// Traditional approach let sqr_arr = []; for(let i = 0; i < nums.length; i++) { sqr_arr.push(nums[i] * nums[i]); } // Modern approach with map() ‚≠ê let sqr_arr = nums.map(x => x * x); console.log(sqr_arr); // [1, 4, 9, 16, 25]

filter() - Select Array Elements

// Filter even numbers let even_arr = nums.filter(x => x % 2 == 0); console.log(even_arr); // [2, 4] // Filter with objects let members = [ {name: "Amit", age: 23}, {name: "Sumit", age: 19}, {name: "Suresh", age: 22}, ]; let allowed_members = members.filter((m) => { return m.age >= 21; }); console.log(allowed_members); // [{name: "Amit", age: 23}, {name: "Suresh", age: 22}]

Array Method Comparison

Method Purpose Returns Use Case
map() Transform elements New array (same length) Square all numbers, format strings
filter() Select elements New array (subset) Get even numbers, filter by age
forEach() Execute function undefined Log each element, update DOM
reduce() Accumulate value Single value Sum, find max/min, group data

‚è±Ô∏è Asynchronous Operations

setTimeout - Delayed Execution

// Execute after 10 seconds (10000ms) setTimeout(() => { console.log("Happy New Year"); }, 10000);

setInterval - Repeated Execution

let cnt = 0; // Execute every second (1000ms) setInterval(() => { console.log(cnt++); }, 1000); // Output: 0, 1, 2, 3, 4, 5, ...
‚ö†Ô∏è Class Work Challenge: How to clear the interval after 5 seconds?
Click to see solution
let cnt = 0; let intervalId = setInterval(() => { console.log(cnt++); }, 1000); setTimeout(() => { clearInterval(intervalId); console.log("Interval cleared!"); }, 5000);

üåê API Integration with Fetch

Fetching Data from GitHub API

const url = "https://api.github.com/users"; fetch(url) .then(response => response.json()) .then(users => { const tbody = document.getElementById("tbody"); users.forEach(user => { tbody.innerHTML += ` <tr> <td>${user.id}</td> <td>${user.login}</td> <td> <img src="${user.avatar_url}" width="50"> </td> </tr> `; }); }) .catch(error => { console.log("Error:", error); });

Understanding Fetch API:
  • fetch(url) returns a Promise
  • First .then() receives the Response object
  • response.json() parses JSON and returns another Promise
  • Second .then() receives the actual data
  • .catch() handles any errors in the chain

React Hooks

useState & useEffect - API Integration (Users.jsx)

import React, { useEffect, useState } from 'react' import axios from 'axios' export default function Users() { // State management const [users, setUsers] = useState([]) // Side effect - runs after component mounts useEffect(() => { axios.get('https://api.github.com/users') .then(response => { setUsers(response.data) }) .catch(error => { console.error(error) }) }, []) // Empty dependency array = run once on mount return ( <div> <table border="1" cellPadding="10"> <thead> <tr> <th>ID</th> <th>Username</th> <th>Avatar</th> </tr> </thead> <tbody> {users.map(user => ( <tr key={user.id}> <td>{user.id}</td> <td>{user.login}</td> <td> <img src={user.avatar_url} alt={user.login} width="50" /> </td> </tr> ))} </tbody> </table> </div> ) }

React Component Lifecycle with Hooks

Key React Concepts Covered:

Concept Purpose Example
Components Reusable UI building blocks Hi, Hello, Greet, Sqr, Users
Props Pass data to components <Sqr num={5} />
useState Manage component state const [users, setUsers] = useState([])
useEffect Handle side effects API calls, subscriptions, timers

map() Render lists {users.map(user => ...)}
üí° axios vs fetch:
  • axios: Automatic JSON parsing, better error handling, request/response interceptors
  • fetch: Native browser API, requires manual JSON parsing, simpler for basic requests
  • Both are valid choices; axios provides more features and convenience
